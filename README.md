# Wiki 2.0 — Документация с поиском на Meilisearch

## Пустой каркас iCore Wiki

Новые окружения создаются без демо-оргструктуры и статей. Базовые таблицы поднимаются из `backend/schema.sql`, после чего backend применяет совместимые миграции при запуске.

Для локальной демонстрации старый набор тестовых данных можно включить явно:

```bash
cd backend
SEED_DEMO_DATA=true npm run dev
```

В production и `docker-compose.yml` демо-наполнение отключено. Существующие записи в уже созданной базе не удаляются.

Администратору доступен отчёт `GET /api/admin/analytics` и вкладка «Аналитика» с динамикой просмотров, активностью сотрудников, статистикой разделов, авторами, устаревшими статьями и CSV-выгрузкой.

Тестовый контент устанавливается отдельной подтверждаемой командой. Она архивирует обычные статьи, сохраняет классификатор (`auto-list*`) и создаёт материалы с префиксом `[Тест]`:

```bash
cd backend
npm run build
CONFIRM_TEST_CONTENT=true npm run seed:test-content
```

Wiki 2.0 — это быстрая и современная база знаний на базе **React / Vite**, **Node.js (Express)**, **PostgreSQL** и поискового движка **Meilisearch**. Данный проект оптимизирован для работы в бесплатных облачных средах (таких как Render Free Plan) за счет легковесной архитектуры и эффективного использования ресурсов.

## 🚀 Почему Meilisearch?

Вместо Elasticsearch в проекте используется **Meilisearch**, что дает следующие преимущества:
* **Минимальное потребление памяти:** Требует всего 50–150 МБ RAM (Elasticsearch требует более 512 МБ–1 ГБ).
* **Мгновенный запуск:** Запуск контейнера занимает доли секунды.
* **Умный поиск из коробки:** Поддержка опечаток (typo tolerance), префиксного поиска по мере ввода (realtime autocomplete) и гибкой настройки релевантности.

### 💡 Совместимость с Render Free Plan
На бесплатном тарифе Render диски для контейнеров являются временными (ephemeral). При каждом перезапуске инстанса данные Meilisearch стираются. Чтобы это не приводило к потере поискового индекса:
* При запуске бэкенд проверяет и автоматически перестраивает индекс Meilisearch, загружая и синхронизируя все опубликованные статьи и документы из постоянной базы данных PostgreSQL.
* Любые действия со статьями (создание, обновление, удаление) синхронизируются с поисковым движком на лету.

## 🛠️ Переменные окружения (Environment Variables)

Для работы поиска необходимо настроить следующие переменные окружения в бэкенде:

| Переменная | Описание | Значение локально |
| --- | --- | --- |
| `MEILI_HOST` | URL-адрес поискового сервера Meilisearch | `http://localhost:7700` |
| `MEILI_MASTER_KEY` | Мастер-ключ авторизации API | `masterKey` |

## 📦 Локальный запуск (Docker Compose)

### 1. Запуск инфраструктуры (База данных и Поиск)
В корне проекта выполните команду для запуска PostgreSQL и Meilisearch:
```bash
docker-compose up -d
```
Это запустит:
* **PostgreSQL** на порту `5432`
* **Meilisearch** на порту `7700` (веб-панель управления доступна по адресу `http://localhost:7700` с паролем `masterKey`)

### 2. Запуск Бэкенда (Backend)
Перейдите в папку бэкенда, установите зависимости и запустите dev-сервер:
```bash
cd backend
npm install
npm run dev
```
Бэкенд запустится на порту `5000`. При запуске произойдет автоматическая проверка соединения и синхронизация всех статей из Postgres в индекс Meilisearch.

### 3. Запуск Фронтенда (Frontend)
Перейдите в папку фронтенда, установите зависимости и запустите приложение:
```bash
cd ../frontend
npm install
npm run dev
```
Фронтенд запустится на порту `3000` (или другом свободном порту, указанном Vite).

## 🔍 Примеры API-запросов (Search API)

Бэкенд предоставляет два эндпоинта для работы с поиском:

### 1. Глобальный поиск с фильтрами и подсветкой
Используется для поиска по заголовкам, краткому содержанию и тексту статей/импортированных документов.

* **Эндпоинт:** `GET /api/search`
* **Параметры запроса:**
  * `q` (string, необязательный) — текст поискового запроса.
  * `category` (string, необязательный) — фильтрация по slug-категории.
  * `tag` (string, необязательный) — фильтрация по тегу.
* **Пример запроса:**
  ```http
  GET http://localhost:5000/api/search?q=заказ&category=new-driver
  ```
* **Пример ответа:**
  ```json
  [
    {
      "id": 1,
      "title": "Не приходят <mark class=\"bg-indigo-500/20 text-indigo-200 px-1 rounded font-semibold\">заказы</mark> — что делать?",
      "slug": "net-zakazov",
      "summary": "Пошаговое руководство по решению проблем с получением заказов в приложении Яндекс Про.",
      "categoryName": "new-driver",
      "tags": ["Заказы", "Техподдержка", "GPS"],
      "published": true,
      "createdAt": "2026-05-29T08:41:01.000Z",
      "highlights": [
        "Если в приложении Яндекс Про долго нет предложений о поездках, проверьте ключевые настройки... Сервис распределяет <mark class=\"bg-indigo-500/20 text-indigo-200 px-1 rounded font-semibold\">заказы</mark> автоматически..."
      ],
      "score": 0.85
    }
  ]
  ```

### 2. Подсказки автодополнения (Autocomplete / Suggestions)
Быстрый поиск до 5 наиболее релевантных статей по мере ввода пользователем символов в поисковую строку.

* **Эндпоинт:** `GET /api/search/suggest`
* **Параметры запроса:**
  * `q` (string, обязательный) — текст для автодополнения.
* **Пример запроса:**
  ```http
  GET http://localhost:5000/api/search/suggest?q=комфорт
  ```
* **Пример ответа:**
  ```json
  [
    {
      "id": 3,
      "title": "Стандарты и правила тарифа «Комфорт»",
      "slug": "standarts-komfort",
      "categoryName": "rides"
    }
  ]
  ```
